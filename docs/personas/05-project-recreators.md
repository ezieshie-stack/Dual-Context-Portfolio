# Persona 5.1: Bootcamp Student

## Demographics
| Attribute | Value |
|-----------|-------|
| **Name** | Alex Rivera |
| **Age** | 24 |
| **Role** | Data Analytics Bootcamp Student |
| **Company** | N/A (In learning phase) |
| **Reports to** | Bootcamp instructor |
| **Education** | BA Psychology, Bootcamp in progress |
| **Location** | Online |

## Short Bio
Alex recently enrolled in a data analytics bootcamp and is looking for real-world project examples to learn from. They want to understand how professionals build portfolio-worthy projects and gain confidence in their abilities.

## Goals
- Learn from real-world examples
- Build impressive portfolio projects
- Understand industry expectations

## Pain Points
- Gap between bootcamp and real work
- All portfolio projects look the same
- Imposter syndrome about abilities

## Scenarios

### Scenario 1: Getting Started
*I found David's portfolio and want to build something similar. I ask "Where do I start with a project like the churn predictor?" Bot gives me a step-by-step roadmap: data collection, feature engineering, modeling, evaluation. I have a clear path.*

### Scenario 2: Prerequisites
*I ask "What do I need to know before building this?" Bot lists prerequisites: SQL basics, Python/Pandas, basic statistics, ML concepts. Now I know what to study first.*

### Scenario 3: Dataset Source
*I ask "Where can I get similar data?" Bot suggests Kaggle Telco churn dataset and PaySim for fraud. I have practice datasets.*

### Scenario 4: Step-by-Step Walkthrough
*I ask "Can you walk me through the process?" Bot breaks down each phase with explanation. Tutorial-style guidance.*

### Scenario 5: Common Mistakes
*I ask "What mistakes should I avoid?" Bot shares lessons about data leakage, overfitting, and feature timing. Learning from experience.*

### Scenario 6: Time Estimate
*I ask "How long will this take me?" Bot gives realistic timeline for a beginner. Set proper expectations.*

### Scenario 7: Portfolio Value
*I ask "Is this good for my portfolio?" Bot explains why this type of project stands out. Confidence boost.*

### Scenario 8: Simpler Version
*I ask "Can I do a simpler version?" Bot suggests starting with fewer features and basic model. Manageable scope.*

### Scenario 9: Resources
*I ask "What should I study first?" Bot provides learning path with priorities. Clear direction.*

### Scenario 10: Getting Help
*I ask "Where can I get help if stuck?" Bot suggests communities, resources, and documentation. Support network.*
